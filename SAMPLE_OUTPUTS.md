## 2. PDF Ingestion

**Request:**
```
POST /ingest
Content-Type: multipart/form-data
file: peblo_pdf_grade1_math_numbers.pdf
```

**Response:**
```json
{
  "document_id": "accfb350-968d-4916-b816-58702ba4be8a",
  "filename": "peblo_pdf_grade1_math_numbers.pdf",
  "grade": 1,
  "subject": "Math",
  "chunk_count": 1,
  "message": "Successfully ingested peblo_pdf_grade1_math_numbers.pdf with 1 chunks"
}
```

**What happens behind the scenes:**
1. PDF text is extracted using PyMuPDF
2. Text is cleaned (whitespace normalized, special chars removed)
3. Grade and subject are detected from filename (`grade1` -> 1, `math` -> "Math")
4. Text is split into chunks (512 chars, 50 overlap, sentence-boundary-aware)
5. Embeddings are generated for each chunk via Ollama
6. Everything is stored in PostgreSQL

**Duplicate detection:** Re-uploading the same PDF returns:
```json
{
  "detail": "This document has already been ingested"
}
```

## 6. Adaptive Difficulty in Action

**Scenario:** Student answers 4 questions correctly in a row (100% accuracy > 80% threshold).

| Answer # | Correct? | Accuracy | Recommended Difficulty |
|----------|----------|----------|----------------------|
| 1        | Yes      | 100%     | easy (< 3 answers, no adjustment) |
| 2        | Yes      | 100%     | easy (< 3 answers, no adjustment) |
| 3        | Yes      | 100%     | **medium** (3+ answers, >80% triggers increase) |
| 4        | Yes      | 100%     | **medium** (stays elevated) |
| 5        | No       | 80%      | easy (accuracy dropped, no longer >80%) |

**Scenario:** Student gets everything wrong (0% accuracy < 40% threshold).

| Answer # | Correct? | Accuracy | Recommended Difficulty |
|----------|----------|----------|----------------------|
| 1        | No       | 0%       | medium (< 3 answers) |
| 2        | No       | 0%       | medium (< 3 answers) |
| 3        | No       | 0%       | **easy** (3+ answers, <40% triggers decrease) |

## 7. Database Schema

```sql
-- source_documents
  id                        uuid                 NOT NULL (PK)
  filename                  character varying    NOT NULL
  content_hash              character varying    NOT NULL (UNIQUE)
  grade                     integer              NULLABLE
  subject                   character varying    NULLABLE
  page_count                integer              NOT NULL
  created_at                timestamp            DEFAULT now()

-- content_chunks
  id                        uuid                 NOT NULL (PK)
  document_id               uuid                 NOT NULL (FK -> source_documents)
  chunk_index               integer              NOT NULL
  topic                     character varying    NULLABLE
  text                      text                 NOT NULL
  embedding                 jsonb                NULLABLE
  created_at                timestamp            DEFAULT now()

-- quiz_questions
  id                        uuid                 NOT NULL (PK)
  chunk_id                  uuid                 NOT NULL (FK -> content_chunks)
  question_type             enum(mcq,true_false,fill_blank)  NOT NULL
  difficulty                enum(easy,medium,hard)           NOT NULL
  question_text             text                 NOT NULL
  options                   jsonb                NULLABLE
  correct_answer            character varying    NOT NULL
  explanation               text                 NULLABLE
  embedding                 jsonb                NULLABLE
  quality_score             double precision     NULLABLE
  is_valid                  boolean              DEFAULT true
  created_at                timestamp            DEFAULT now()

-- student_answers
  id                        uuid                 NOT NULL (PK)
  question_id               uuid                 NOT NULL (FK -> quiz_questions)
  student_id                character varying    NOT NULL
  submitted_answer          character varying    NOT NULL
  is_correct                boolean              NOT NULL
  answered_at               timestamp            DEFAULT now()
```

## 9. Validation Examples

### Input validation:
```
POST /generate-quiz with {"difficulty": "impossible"}
-> 400: {"detail": "difficulty must be one of: {'easy', 'medium', 'hard'}"}

POST /submit-answer with {"selected_answer": "  "}
-> 400: {"detail": "Answer cannot be empty"}

POST /ingest with duplicate PDF
-> 409: {"detail": "This document has already been ingested"}

POST /ingest with non-PDF file
-> 400: {"detail": "Only PDF files are accepted"}
```

### Question quality validation:
Each generated question goes through:
1. **Structural check** - MCQ has 4 options? True/False answer is "True"/"False"? Fill-blank has "___"?
2. **LLM quality scoring** - Rates clarity, correctness, difficulty calibration, educational value (0.0 to 1.0)
3. Questions scoring below 0.4 are rejected

Sample quality scores from Math quiz:
- "How many sides does a triangle have?" -> **0.9** (clear, correct, age-appropriate)
- "A square has four equal sides." -> **1.0** (perfect true/false question)
- "Which number comes after 19?" -> **0.8** (good recall question)
